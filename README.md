# Interface e Jornada do Usuário – Atividade de Wireframe

## 📚 Sobre a disciplina

A disciplina **Interface e Jornada do Usuário** aborda conceitos fundamentais de **UX (User Experience)** e **UI (User Interface)**, com foco em como usuários interagem com sistemas digitais.

Durante a matéria são estudados temas como:

* Experiência do usuário
* Arquitetura da informação
* Fluxos de navegação
* Prototipação
* Testes com usuários
* Design centrado no usuário

O objetivo é desenvolver interfaces que sejam **claras, eficientes e intuitivas**, facilitando a realização de tarefas dentro de aplicativos ou sistemas.

---

# 📝 Descrição da atividade

Nesta atividade foi proposto o desenvolvimento de um **wireframe mobile de baixa fidelidade**, utilizando o software **Figma**, a partir de uma situação-problema realista.

O projeto deve considerar:

* **Tarefa principal do usuário**
* **Jornada de uso**
* **Arquitetura da informação**
* **Clareza funcional**
* **Flexibilidade de uso**
* **Validação com usuários**

Além disso, o fluxo do aplicativo deve permitir **pelo menos dois caminhos diferentes para que o usuário consiga alcançar o mesmo objetivo**.

---

# 🎯 Objetivo do projeto

Criar a estrutura de um aplicativo mobile que permita ao usuário **marcar consultas médicas ou acessar serviços de saúde**, garantindo uma experiência simples e intuitiva.

A proposta demonstra como diferentes usuários podem resolver a mesma necessidade por caminhos distintos dentro da interface.

---

# 🏥 Situação-problema

Usuários precisam **obter atendimento médico**, podendo escolher entre duas opções:

* Consulta presencial em uma clínica
* Consulta online (telemedicina)

O aplicativo deve permitir que o usuário escolha o tipo de atendimento e conclua o agendamento.

---

# 👤 Tarefa principal do usuário

Agendar um atendimento médico.

---

# 🧭 Jornada do usuário

## Caminho 1 – Consulta presencial

Fluxo de navegação:

Home
→ Consulta presencial
→ Escolher especialidade
→ Escolher médico
→ Selecionar data e horário
→ Confirmar consulta

---

## Caminho 2 – Consulta online

Fluxo de navegação:

Home
→ Consulta online
→ Escolher especialidade
→ Selecionar horário disponível
→ Confirmar consulta
→ Entrar na chamada de vídeo

---

# 🧠 Arquitetura da informação

Estrutura básica do aplicativo:

Home
├── Consulta presencial
│   ├── Especialidade
│   ├── Médico
│   ├── Horário
│   └── Confirmação
│
└── Consulta online
├── Especialidade
├── Horário
└── Confirmação

Outras áreas do aplicativo incluem:

* Minhas consultas
* Perfil do usuário

---

# 📱 Estrutura das telas (Wireframe)

O protótipo contém as seguintes telas principais:

1. Tela inicial (Home)
2. Seleção de tipo de consulta
3. Escolha de especialidade
4. Lista de médicos
5. Perfil do médico
6. Seleção de data
7. Seleção de horário
8. Confirmação da consulta
9. Tela de consulta online
10. Tela de confirmação

Todos os wireframes foram desenvolvidos em **baixa fidelidade**, focando na estrutura da interface e no fluxo de navegação.

---

# 🧪 Validação com usuários

Para avaliar a clareza do fluxo de navegação, foram realizados testes simples com usuários.

Resultados observados:

* Usuários conseguiram compreender facilmente a diferença entre consulta presencial e online.
* A maioria conseguiu completar o agendamento sem dificuldades.
* O fluxo de navegação foi considerado simples e intuitivo.

Sugestões de melhoria incluíram:

* Destacar melhor os botões principais na tela inicial.
* Simplificar algumas etapas do agendamento.
